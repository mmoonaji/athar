# محرك توليد المحتوى بالذكاء الاصطناعي — أثر (AI Content Generation Engine)

يوضح هذا الدليل الهيكل البرمجي والمحركات المسؤولة عن أتمتة توليد الدروس الفقهية والعقدية في منصة أثر باستخدام الذكاء الاصطناعي، مع ضمان الجودة العلمية والتطابق مع مخططات النظام الفنية.

---

## 1. هيكلية ومسار عمل المحرك (Prompt & Generation Flow)

يتبع المحرك مسار عمل خطي متسلسل يفصل المسؤوليات لضمان أقصى درجات الضبط العلمي:

```
[مواصفة الدرس] ➔ [Prompt Builder] ➔ [معالج محرك الذكاء الاصطناعي] ➔ [مسودة الدرس] ➔ [مراجعة وتدقيق علمي] ➔ [تحويل الهيكل الفني] ➔ [التحقق] ➔ [الاستيراد]
```

1. **Prompt Builder**: يقوم بقراءة مواصفة الدرس (`LessonSpec`) ويدمجها ديناميكياً مع موجهات الكتابة المحددة مسبقاً في مكتبة الموجهات (`content/prompts/writer.md`).
2. **محرك التوليد (AI Driver)**: يعتمد المحرك على واجهة برمجية موحدة (`AiAdapter`) تدعم التبديل السلس بين مزودي الخدمة المختلفين:
   - **Gemini**: عبر `GeminiAdapter`
   - **OpenAI**: عبر `OpenAIAdapter`
   - **Claude**: عبر `ClaudeAdapter`
   - **DeepSeek**: عبر `DeepSeekAdapter`
3. **التدقيق الشرعي واللغوي (Review Flow)**: يُمرر المحتوى المتولد على مدقق التدقيق العلمي (`scripts/review-lesson.ts`) والذي ينشئ تقريراً باللغة العربية حول مطابقة الآيات والأحاديث وخلو النص من الأخطاء العقدية والنحوية.
4. **تحويل الهيكل (JSON Conversion)**: تُحوّل المسودة المدققة آلياً إلى المخطط الفني المعتمد (`scripts/convert-lesson.ts`).
5. **التحقق النهائي والرفع**: يمر الدرس على طبقة التحقق النهائي (`src/lib/ai/output-validator.ts`) للتأكد من حجم الكلمات وأعداد الأسئلة وصحة هيكل الخيارات قبل حفظه في المسار النهائي للرفع.

---

## 2. هيكل ومسؤوليات المجلدات (Folder Responsibilities)

تتوزع الملفات في مجلد `content/` وفق التقسيم التالي لمنع خلط المراحل:

- **`content/prompts/`**: يحتوي على القوالب والتعليمات الإرشادية لكل دور ذكاء اصطناعي (الكاتب، المراجع، مصمم الاختبارات، محول الهيكل الفني).
- **`content/specifications/`**: يحتوي على مواصفات الدروس الأولية المحددة للدروس قبل كتابتها.
- **`content/drafts/`**: لحفظ مخرجات التوليد الأولية الخام (المسودات النصية) قبل التدقيق.
- **`content/reviews/`**: يحتوي على تقارير المراجعة العلمية والشرعية واللغوية بصيغة Markdown.
- **`content/reports/`**: لحفظ تقارير التوليد والتحقق بصيغة JSON لسهولة الكشف التلقائي عن جودة التوليد الإجمالية.
- **`content/lessons/`**: المجلد النهائي للدروس المطابقة للمخطط الفني بالكامل والجاهزة للاستيراد لقاعدة البيانات.

---

## 3. كيفية التشغيل والاختبار

### تشغيل توليد الدرس من المواصفة:
يقوم بقراءة المواصفة، دمجها في الموجه، توليد المسودة وحفظها وحفظ تقارير التوليد الفني.
```bash
npx tsx scripts/generate-lesson.ts content/specifications/example.json
```

### تشغيل التدقيق العلمي والشرعي للدرس المولد:
يفحص الكتل والمراجع والآيات ومطابقتها ويكتب تقريراً فصيلاً في مجلد التقارير.
```bash
npx tsx scripts/review-lesson.ts content/lessons/example.json
```

### تشغيل تحويل المسودة النصية إلى JSON معتمد فنيّاً:
```bash
npx tsx scripts/convert-lesson.ts content/drafts/example-lesson-spec-draft.txt
```
