# نظام إدارة المحتوى للمشرفين — أثر (Athar CMS Documentation)

يوضح هذا الدليل الهيكل البرمجي والتشغيلي لنظام إدارة المحتوى الداخلي المخصص للمشرفين والمدققين في منصة أثر (أثر).

---

## 1. الهيكل البرمجي ونموذج الصلاحيات (Architecture & Permissions)

تم بناء نظام إدارة المحتوى الداخلي باستخدام هيكل الميزات المعزول (`src/features/admin/`):
- **المكونات التفاعلية**: توجد تحت مجلد `components/` مستقلة تماماً ومفصولة عن الواجهة العامة للمتعلمين.
- **إجراءات الخادم (Server Actions)**: تقع تحت مجلد `actions/` وتتحقق صارماً من صلاحية الحساب الطالب للإجراء.

### نموذج صلاحيات المشرفين (Permission Model):
- **حماية الواجهة**: تتم حماية مسار اللوحة `/admin` بالكامل في ملف التوجيه والتحقق `src/proxy.ts`؛ يتم التحقق من الجلسة ومطابقة معرّف المستخدم مع الحقل `role` في جدول `public.profiles`.
- **المنع (403 Forbidden)**: في حال محاولة مستخدم عادي الدخول، تُرجع اللوحة رمز الاستجابة `403` وصفحة منع واضحة.
- **حماية الإجراءات**: تقوم جميع Server Actions باستدعاء الدالة المساعدة `assertAdmin()` التي تتحقق من هوية المستخدم ودوره كـ `ADMIN` قبل تنفيذ أي استعلام أو كتابة في قاعدة البيانات.

---

## 2. مسار النشر والاعتماد (Publishing Workflow)

يمر الدرس بـ ٦ حالات متسلسلة للتحكم في مراحل بناء المنهج:

1. **المواصفة (SPECIFICATION)**: تعريف الدرس فنيّاً وتحديد أهدافه الشرعية والكلمات الدلالية.
2. **المسودة (DRAFT)**: توليد المسودة النصية للدرس عبر المكون الذكي المقترن.
3. **التدقيق الشرعي (REVIEWED)**: إتمام المراجعة والتدقيق العلمي والأخلاقي وكتابة الملحوظات والتوجيهات.
4. **التحقق الفني (VALIDATED)**: تمرير ملف المحتوى على مدقق الكتل الفنية والـ JSON لضمان خلوه من التعارضات أو تكرار المعرفات.
5. **المنشور (PUBLISHED)**: رفع الدرس لقاعدة البيانات وإتاحته للجمهور بشكل فوري.
6. **المؤرشف (ARCHIVED)**: إزالة الدرس من القائمة العامة مع الاحتفاظ به في الأرشيف للرجوع التاريخي.

## 3. بنية الملفات والمجلدات (Folder Structure)

تتوزع مكونات لوحة التحكم كالتالي:

```
src/
 ├─ app/
 │   └─ admin/                    # مسارات اللوحة والصفحات الفرعية
 │       ├─ layout.tsx            # التخطيط العام وشريط التنقل والمسار
 │       ├─ page.tsx              # الصفحة الرئيسية لعدادات وإحصاءات اللوحة
 │       ├─ curriculum/           # إدارة شجرة وهيكلة المناهج
 │       ├─ lessons/              # جدول إدارة وحذف ونشر الدروس
 │       ├─ specifications/       # التحكم بخط إنتاج المحتوى التلقائي
 │       ├─ reviews/              # تدقيق المسودات وتقارير الاعتماد
 │       └─ settings/             # تعديل ثيمات اللوحة ومزود الذكاء الاصطناعي
 └─ features/
     └─ admin/                    # مجلد الميزات المعزول للمشرفين
         ├─ components/           # المكونات التفاعلية والواجهات
         ├─ actions/              # إجراءات الخادم الآمنة (Server Actions)
         └─ types/                # تعريفات وأنواع هياكل البيانات المقترنة
```

## 4. التوسعات المستقبلية (Future Extensions)

1. **مكتبة الوسائط الكاملة (Media Library)**: ربط اللوحة بـ Supabase Storage لتمكين المشرفين من رفع الصور التوضيحية وملفات التجويد الصوتية.
2. **لوحات التحليل البياني (Analytics)**: استبدال العدادات الحالية برسوم بيانية تفاعلية لقياس سلوك إتمام الدروس وساعات ذروة التفاعل.
3. **تعدد المحررين (Multi-Editor Support)**: توسيع جدول الأدوار لدعم مدققين شرعيين بصلاحيات تدقيق فقط بدون صلاحيات حذف أو تعديل إعدادات النظام الأساسية.
